# FAQ 知识库查询技能

## 描述

从 `knowledge/faq/` 中检索最相关的条目，用于回答产品使用、接入、支持渠道、工单流程等常见问题。

## 知识库结构

```
knowledge/faq/
├── general.md      # 通用说明、平台支持、人工客服、处理时效
├── account.md      # 账户、登录、安全处理
├── billing.md      # 计费、退款、发票
├── technical.md    # 启动、配置、接口、故障排查
└── ...
```

## 查询流程

1. 识别用户问题的关键词、意图和语言。
2. 优先搜索最可能的 FAQ 文件，再扩展到整个 `knowledge/faq/` 目录。
3. 若命中多个条目，合并成一个简洁回答，避免逐段复制。
4. 若没有足够依据，不要猜测，转入建单或人工流程。

## 匹配策略

- **精确匹配**：问题文本直接命中 FAQ 标题或关键短语。
- **语义匹配**：问题表达不同，但意图一致，例如“机器人怎么跑起来”与“如何启动 bot”。
- **多条目合并**：一个问题同时涉及配置和工单时，可合并多个 FAQ 回答。

## 回复模板

命中 FAQ 时：

```text
您好，关于您的问题，当前可参考以下信息：

[整理后的答案]

如果您愿意，我也可以继续帮您排查或为您创建工单。
```

未命中 FAQ 时：

```text
抱歉，我暂时没有在知识库中找到足够准确的答案。为了避免误导您，我建议现在为您创建工单，由人工继续跟进。
```

## 升级条件

- FAQ 信息明显不足以支持结论
- 用户提问涉及故障、投诉、数据异常或时效承诺
- 问题连续两轮仍未解决

## 注意事项

- 只回答知识库中明确记录的内容
- 不编造产品链接、政策或处理时效
- 如果 FAQ 可能过时，提醒用户以最新官方通知为准
- 回答后优先说明下一步，而不是只给静态信息
